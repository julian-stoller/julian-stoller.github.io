Sort Everything

A Python desktop application for organizing and analyzing large media collections — images, videos, and documents — with XMP metadata, AI classifiers, face recognition, and more. Runs entirely locally.

Overview

Sort Everything started as a personal tool for managing a large photo library and grew into something considerably more involved. The core idea is simple: tag files, filter by those tags, and keep everything organized. The implementation is not.

Under the hood it combines a SQLite database, XMP metadata written directly into file binaries, AI classifiers trained on user-tagged data, perceptual hashing for duplicate detection, face recognition via DeepFace, YOLOv8 pose estimation, background OCR, and a VLC-based in-app video player — all tied together in a Tkinter GUI and running entirely on the local machine. No cloud services, no subscriptions, no accounts.

Everything stays local

All processing — inference, training, OCR, face detection — runs on the local machine. Tags are embedded directly into files using XMP metadata, so they travel with the files and remain readable by tools like Adobe Lightroom and Photoshop even outside the app.

Technical Features

XMP Metadata Tagging

Tags are embedded directly into files as XMP metadata, not stored only in a sidecar database. For images (JPEG, PNG, TIFF, WebP, HEIC), this is done through pure Python binary manipulation of the file structure. For video (MP4, MOV), it uses ExifTool integration. This means tags survive moves, renames, and copies, and remain accessible to any XMP-aware application.

Virtual Collections (PseudoFolders)

A lightweight "smart playlist" system for grouping files without moving them. Each collection tracks multiple source folders with per-folder inclusion and exclusion rules. Collections are persisted as JSON with automatic schema migration across versions — useful as the data model evolved significantly over time.

SQLite Database & Perceptual Hashing

All file records, tags, and metadata are stored in SQLite. Perceptual hashing (pHash) is computed for images to support duplicate detection and similar-file grouping. Background monitoring watches tracked folders for new files and automatically runs similarity checks as new content arrives.

AI Image Classification

Custom binary and multi-class image classifiers are trained directly from tagged files in the collection. The training pipeline includes:

  • A training queue with priority scheduling
  • Auto-retrain triggers when a model becomes stale relative to its training data
  • Data gap and class imbalance tracking
  • Live training metrics visible during training (accuracy, validation AUC)

The result is a growing set of classifiers that improve as the collection gets more tagged examples — without any external API calls or GPU requirements beyond what's available locally.

Face Recognition

DeepFace integration handles face detection and embedding across the collection, enabling browsing and filtering by detected faces. Face data is stored alongside file records in the database.

Pose Estimation & OCR

YOLOv8-based pose estimation runs on images to identify human poses. Background OCR processing extracts text from images and documents, with results indexed for search. Both run as low-priority background tasks to avoid blocking the UI during normal use.

Video Playback

An in-app video player built on VLC handles playback directly within the application. Resource management was a design challenge here — VLC's Python bindings require careful lifecycle handling to prevent memory leaks across many files.

Batch Operations & Tag Filtering

Complex filter expressions support browsing by any combination of tags, with AND/OR/NOT logic. Batch tagging queues allow applying tags across large groups of files asynchronously, with priority scheduling so interactive operations stay responsive.
