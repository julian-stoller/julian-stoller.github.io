Sort Everything

A Python desktop application for organizing and analyzing large media collections — images, videos, and documents — with XMP metadata, AI classifiers, face recognition, and more. Runs entirely locally.

Overview

Sort Everything started as a personal tool for managing a large photo library and grew into something considerably more involved. The core idea is simple: tag files, filter by those tags, and keep everything organized. The implementation is not.

Under the hood it combines a SQLite database, XMP metadata written directly into file binaries, AI classifiers trained on user-tagged data, perceptual hashing for duplicate detection, face recognition via DeepFace, YOLOv8 pose estimation, background OCR, and a VLC-based in-app video player — all tied together in a Tkinter GUI and running entirely on the local machine.

Everything stays local

All processing — inference, training, OCR, face detection — runs on the local machine. Tags are embedded directly into files using XMP metadata, so they travel with the files and remain readable by tools like Adobe Lightroom and Photoshop even outside the app.

Browsing & Tag Filtering

The main view is a filterable file list. Filter expressions support arbitrary combinations of tags — AND, OR, exclusion — so drilling into any specific subset of a large collection is fast. Suggested tags appear based on what's already applied to the current filtered set.

Compound filters work the same way — any number of inclusion and exclusion rules can be combined. The result count updates immediately, making it straightforward to explore a collection by multiple criteria at once.

File Detail & Face Recognition

Clicking into any file opens a detail view with the full-size image, current tags, file metadata, and a perceptual hash. Tags can be added, removed, or applied in bulk from this view.

DeepFace integration handles face detection and embedding across the collection. Detected faces are surfaced in the detail view as cropped thumbnails alongside the detection result, and stored in the database for collection-wide browsing by face.

AI Image Classification

Custom binary and multi-class image classifiers are trained directly from tagged files in the collection. The more a collection gets tagged, the more useful the classifiers become — without any external API calls or GPU requirements beyond what's available locally.

Training pipeline

  • A priority-based training queue manages multiple classifier jobs
  • Auto-retrain triggers fire when a model becomes stale relative to new tagged data
  • Data gap and class imbalance tracking surfaces when a classifier needs more examples
  • Training metrics are plotted live as epochs complete

XMP Metadata & File Portability

Tags are embedded directly into files as XMP metadata, not stored only in the database. For images (JPEG, PNG, TIFF, WebP, HEIC), this is done through pure Python binary manipulation of the file structure. For video (MP4, MOV), it uses ExifTool. Tags survive moves, renames, and copies, and remain accessible to any XMP-aware application.

Virtual Collections, Database & Background Processing

Virtual Collections (PseudoFolders)

A lightweight "smart playlist" system for grouping files without moving them. Each collection tracks multiple source folders with per-folder inclusion and exclusion rules, persisted as JSON with automatic schema migration across versions.

SQLite & Perceptual Hashing

All file records, tags, and metadata are stored in SQLite. Perceptual hashing (pHash) is computed for images to support duplicate detection and similar-file grouping. Background monitoring watches tracked folders for new files and automatically runs similarity checks as new content arrives.

Pose Estimation, OCR & Video Playback

YOLOv8-based pose estimation runs on images to identify human poses. Background OCR extracts text from images and documents. Both run as low-priority background tasks to avoid blocking the UI. An in-app video player built on VLC handles playback directly within the application, with careful resource lifecycle management to prevent memory leaks across many files.
